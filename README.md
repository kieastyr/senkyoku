# 🎼 選曲・提出システム (Senkyoku)

Python (Streamlit) ベースの選曲・提出管理システムです。
Google スプレッドシートと連携して、選曲データの取得、バリデーション、および提出管理を行います。

## 主な機能
- **セキュアな認証**: `bcrypt` によるハッシュ化されたパスワード認証。
- **メイン曲・サブ曲の選択**: 
  - メイン曲を1曲、サブ曲を最大3曲まで選択可能。
  - 合計分数が100分を超えないサブ曲のみを自動で提示する動的フィルタリング。
- **詳細なバリデーション**:
  - **合計分数**: 0分超、100分以内。
  - **楽器構成**: 各パートの合計奏者数が規定の範囲内（最小〜最大）に収まっているかリアルタイムで判定。
    - 例: フルート/オーボエ/クラ/ファゴット/トランペット (4〜20名)、ホルン (8〜20名)、トロンボーン (3〜20名)、チューバ (1〜20名)。
- **Google Sheets 連携**: 
  - 楽曲リスト（メイン・サブ）をスプレッドシートから動的に読み込み。
  - 提出されたデータをスプレッドシートへ自動保存。
- **コメント表示**: 楽曲データに含まれる「提出者からのコメント」をアプリ上で確認可能。

## セットアップ

### 1. 依存関係のインストール
`uv` を推奨します。
```bash
uv sync
```

### 2. スプレッドシートの準備
1. **提出先シート**: 以下の見出しを1行目に設定してください。
   `タイムスタンプ`, `提出者名`, `メイン曲`, `サブ曲1`, `サブ曲2`, `サブ曲3`, `合計分数`, `楽器構成`, `自由記入欄`
2. **楽曲リストシート**: 「メイン曲」「サブ曲」の2つのワークシートを用意してください。

### 3. 環境設定
`.streamlit/secrets.toml` を作成し、認証情報と Google スプレッドシートの接続情報を設定してください。
```toml
[auth]
id = "YOUR_ID"
password_hash = "YOUR_BCRYPT_HASH"

[connections.gsheets]
spreadsheet = "SUBMISSION_SHEET_URL"
input_list = "SONG_LIST_SHEET_URL"
# サービスアカウント情報
type = "service_account"
...
```

### 4. 実行
```bash
uv run streamlit run main.py
```

## データソースについて
楽曲リストは Google Sheets Connection を介して取得されます。ローカルの `datasource/` ディレクトリはフォルダ構造維持のために `.gitkeep` のみが管理されています。
